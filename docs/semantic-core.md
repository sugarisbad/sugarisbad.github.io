# Семантическое ядро сайта

Кластеры запросов → какая страница их ловит. Ядро уже внесено в title, description
и тексты страниц (`src/i18n/ru.ts`, `src/i18n/en.ts`).

Частотности указаны **оценочно** (порядок величины по нише на 2026 год). Точные цифры
проверяйте в [Вордстате](https://wordstat.yandex.ru) (нужен вход в Яндекс): вводите запрос,
смотрите колонку «Показов в месяц» по нужному региону. Кавычки `"запрос"` дают частоту
без хвостов — она честнее для оценки спроса.

Легенда: 🔥 >5 000 показов/мес · ⭐ 500–5 000 · ✦ <500 (низкочастотка — самая конверсионная).

## Как это внесено в сайт

- **Title / description** каждой страницы начинаются с самых частотных коммерческих
  запросов кластера (см. `home.title`, `servicesPage.title` и т.д. в `src/i18n/*.ts`).
- **Видимые тексты** (подзаголовок hero, лиды секций, названия услуг) содержат ключи
  естественно: «настройка и администрирование Linux-серверов (VPS/VDS)», «server setup
  and hardening» — без переспама, Яндекс наказывает за него («Баден-Баден»).
- **FAQ на /work/** размечен схемой FAQPage — вопросные запросы дают расширенный сниппет.
- Инфозапросы уровня «docker compose как пользоваться» лендингом не ловятся — под них
  нужен блог (см. «что дальше» в README).

## Поддержка ядра

1. Раз в квартал: Вебмастер → «Поисковые запросы» и GSC → «Performance» — смотреть,
   по каким запросам показываетесь на 5–20 позиции, и усиливать их в текстах.
2. Новые услуги — сразу подбирать кластер в Вордстате и добавлять сюда.
3. Плотность: ключ в title + 1–2 раза в тексте страницы достаточно.
