# Бот приёма заявок

Telegram-бот — «приёмная» opsmith: принимает заявки с лендинга, показывает
каталог услуг с ценами и FAQ, ведёт учёт заявок и даёт клиенту живой чат с
инженером прямо в боте.

## Что умеет

**Для клиента:**

- **Главное меню** (`/start` без deep-link): заявка, услуги и цены, FAQ,
  «мои заявки», вопрос инженеру, кнопки на сайт и Calendly.
- **Каталог услуг** (`/services`) — 8 направлений, цены совпадают с прайсом
  сайта (`src/data/pricing.ts`); из карточки услуги оформляется заявка в один тап.
- **FAQ** (`/faq`) — те же вопросы-ответы, что и на сайте.
- **Форма заявки** — имя → задача → телефон → почта, в конце предпросмотр
  с кнопками «✏️ исправить» и «✅ отправить» (плюс защита от даблкликов).
- **Мои заявки** (`/status`) — номера и статусы своих заявок; при смене статуса
  приходит уведомление.
- **Чат с инженером** (`/ask`) — вопросы пересылаются владельцу; если настроена
  Ollama, клиент сразу получает предварительный ответ ИИ, пока инженер не
  ответил лично.

**Для владельца (в ADMIN_CHAT_ID):**

- Под каждой заявкой кнопки **✍️ Ответить / 🔧 В работу / ✅ Закрыта** — ответ
  уходит клиенту от имени бота, смена статуса пишется в базу и уведомляет клиента.
- `/stats` — сводка: всего заявок, за 7 дней, по статусам.

Заявки хранятся в SQLite (`leads.db` рядом с ботом, путь настраивается
через `LEADS_DB`).

Опционально: если на сервере есть [Ollama](https://ollama.com), бот прогоняет
текст заявки через локальную LLM и прикладывает разбор — суть задачи, подходящая
услуга, срочность и уточняющие вопросы.

## Запуск

1. **Создайте бота**: напишите [@BotFather](https://t.me/BotFather) → `/newbot` →
   придумайте имя и юзернейм (например `sugarisbad_orders_bot`) → получите токен.
2. **Узнайте свой chat id**: спросите у [@userinfobot](https://t.me/userinfobot).
3. **Запустите**:

   ```bash
   cd bot
   python -m venv venv && . venv/bin/activate     # Windows: venv\Scripts\activate
   pip install -r requirements.txt
   cp .env.example .env                            # впишите BOT_TOKEN и ADMIN_CHAT_ID
   export $(grep -v '^#' .env | xargs)             # или задайте переменные вручную
   python bot.py
   ```

4. **Проверьте**: напишите боту `/start`, пройдите форму — заявка должна
   прийти вам в личку.

## Деплой на сервер (systemd) — одной командой

```bash
git clone https://github.com/sugarisbad/sugarisbad.github.io.git site
cd site/bot
sudo ./install.sh        # спросит BOT_TOKEN и ADMIN_CHAT_ID, если нет .env
```

Скрипт создаёт служебного пользователя `tgbot`, ставит бота в `/opt/tgbot`
с виртуальным окружением, устанавливает systemd-юнит и запускает сервис.
Повторный запуск скрипта обновляет `bot.py` и перезапускает сервис,
не трогая `.env`.

Управление:

```bash
systemctl status tgbot        # состояние
journalctl -u tgbot -f        # живые логи
sudo systemctl restart tgbot  # перезапуск
```

<details>
<summary>Ручная установка (если не хочется скриптом)</summary>

```bash
sudo useradd -r -s /usr/sbin/nologin tgbot
sudo mkdir -p /opt/tgbot && sudo cp bot.py requirements.txt .env /opt/tgbot/
cd /opt/tgbot && sudo python3 -m venv venv && sudo venv/bin/pip install -r requirements.txt
sudo chown -R tgbot:tgbot /opt/tgbot
sudo cp tgbot.service /etc/systemd/system/
sudo systemctl daemon-reload && sudo systemctl enable --now tgbot
```

## Отслеживание источника заявки (deep-links)

Каждая кнопка «Заказать» на сайте ведёт в бота со своим кодом:
`t.me/<бот>?start=vps-setup`, `?start=plan-ai`, `?start=src-cases` и т.д.
Бот расшифровывает код (словарь `SOURCES` в `bot.py`) и добавляет в заявку
строку «🔗 Откуда: …» — видно, какая услуга или страница привела клиента.
Если добавляете услугу на сайт — добавьте её слаг и в `SOURCES`.

## Подключение к сайту

После создания бота впишите ссылку на него в `src/data/site.ts`:

```ts
botUrl: 'https://t.me/<юзернейм_бота>',
```

На странице появится кнопка «🤖 Оставить заявку в боте» в блоке контактов.

## Лиды в EspoCRM (опционально)

Если у вас развёрнута [EspoCRM](https://www.espocrm.com), бот может создавать
лида из каждой заявки. Добавьте в `.env`:

```
ESPOCRM_URL=https://crm.example.com
ESPOCRM_API_KEY=<ключ>
```

Ключ создаётся в CRM: Администрирование → API Users → создать пользователя
с ролью, у которой есть право `create` на сущность Lead (метод аутентификации
API Key). В лида попадают имя, телефон, почта, Telegram клиента, источник
заявки и текст задачи. Если CRM недоступна, заявка всё равно приходит в
Telegram — доставка от CRM не зависит.

## LLM-разбор (опционально)

На сервере с Ollama добавьте в `.env`:

```
OLLAMA_URL=http://127.0.0.1:11434
OLLAMA_MODEL=gemma2
```

С Ollama бот дополнительно отвечает на вопросы клиентов в чате «вопрос
инженеру» — присылает черновик ответа ИИ, пока владелец не ответил лично.

Если Ollama недоступна, бот просто пришлёт заявку без блока «Разбор» и не
будет присылать черновики — доставка заявок и чат никогда не зависят от LLM.
